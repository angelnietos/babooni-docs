## Objetivo

Mapear el dominio legacy a paquetes **`@ideauto/*`** bajo `libs/clientes/ideauto/`, reusando `@base/*` donde toque. **Cero** paquetes `@saas/*` para este producto.

---

## Principio de colocación

| Pregunta | Respuesta |
|----------|-----------|
| ¿Kernel genérico (auth, users, audit, clients CRUD)? | **`@base/*`** |
| ¿Negocio recalls Ideauto (campañas, oleadas, DGT, VINs)? | **`@ideauto/{domain}-*`** en `libs/clientes/ideauto/` |
| ¿Plantilla demo? | Nunca `@arquetipos/*` |
| ¿SaaS Verifactu? | Nunca `@saas/*` desde Ideauto |

**Por qué cliente y no SaaS:** Ideauto es un **cliente** del monorepo (como Josanz). Recalls es su producto. Ver [nuevo-cliente-checklist.md](../../../clientes/nuevo-cliente-checklist.md).

---

## Mapping

| Legacy | Destino | Paquetes |
|--------|---------|----------|
| Users, login, password, profiles | `@base` + wiring Ideauto | `@base/users-*`, guards Nest |
| Clients, ContactPersons | `@base` (+ extensión Ideauto) | `@base/clients-*` |
| Campaigns, CampaignVehicles, Typologies, History | `@ideauto` | `@ideauto/campaigns-{api,data-access,shell,features}` + BE |
| Waves, Letters | `@ideauto` | Subdominio campaigns **o** `@ideauto/waves-*` |
| Budgets | `@ideauto` | `@ideauto/budgets-*` |
| Invoices, Certificates | `@ideauto` | `@ideauto/invoices-*` / certificados producto (**≠** Verifactu) |
| Dgt* | `@ideauto` | `@ideauto/dgt-*` (SOAP solo aquí) |
| FileStructures, filesApi | `@ideauto` | `@ideauto/files-*` |
| Notes | `@ideauto` | notes lib o feature campaigns |
| RentingCompanies, reports | `@ideauto` | `@ideauto/reports-*` |
| IdeautoReferences, VinsInactive*, vehicleInfo | `@ideauto` | campaigns / vehicles |
| adminHome, dashboard | `@ideauto` | admin shell + features |
| tasks (node-schedule) | worker Ideauto | app worker bajo `clientes/ideauto` |

`Budgets`, `Campaigns`, `CampaignsHistory`, `CampaignTypologies`, `CampaignVehicles`, `Certificates`, `Clients`, `ContactPersons`, `DgtAddresses`, `DgtDeregistrations`, `DgtNotFound`, `DgtRecall`, `DgtRecallNotification`, `DgtReferences`, `FileStructures`, `IdeautoReferences`, `Invoices`, `Letters`, `Notes`, `RentingCompanies`, `Users`, `VinsInactiveNonCampaign`, `Waves`.

Tags: `layer:clientes`, `runtime:*` acorde al path.

---

## Anti-patrones

| No hacer | Por qué |
|----------|---------|
| Poner libs en `libs/productos-saas/` | Capa equivocada |
| Importar `@saas/*` o `@josanz/*` | Boundaries cliente |
| Copiar atomic design como arquitectura | No es el contrato de dominio |
| Endpoints usuarios públicos | Seguridad |

## Criterios

- [ ] Confirmar `waves` sub-feature vs lib 4-capas.
