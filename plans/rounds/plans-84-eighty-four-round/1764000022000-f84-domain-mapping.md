## Objetivo

Mapear **1:1** el dominio legacy a paquetes del monorepo: qué se **reusa** de `@base/*`, qué se **crea** en `@saas/*`, y qué **no** se copia.

---

## Principio de colocación

| Pregunta | Respuesta |
|----------|-----------|
| ¿Es kernel genérico (auth, users, audit, clients CRUD)? | **`@base/*`** — extender, no fork |
| ¿Es negocio recalls (campañas, oleadas, DGT, VINs)? | **`@saas/recalls-*` / `@saas/{domain}-*`** bajo `libs/productos-saas/recalls/` |
| ¿Es plantilla demo? | **Nunca** `@arquetipos/*` desde producto |
| ¿Hace falta wrapper UI por marca? | Solo si aporta valor ([ui-re-export-vs-wrapper](../../../guides/ui-re-export-vs-wrapper.md)) |

**Por qué SaaS y no `clientes/josanz`:** Recalls es producto vendible independiente del ERP Josanz (misma regla que Verifactu). Ver [productos-saas-extends-base.md](../../../productos-saas/productos-saas-extends-base.md).

## Anti-patrones a **no** migrar

| Legacy | Por qué no se copia |
|--------|---------------------|
| Carpetas `atoms/molecules/organisms` como arquitectura de dominio | Es UI kit, no capas de producto |
| `repositories/` + `repository/` duplicados | Un puerto por dominio |
| `queryBuilder.js` genérico | Prisma tipado |
| Redux slices de UI global (`tableState`, `dataRow`) | Estado local / facades |
| Auth “abre usuario sin token” | Violación ADR 0005 / seguridad |
